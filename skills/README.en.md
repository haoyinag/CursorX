# Skills index

`skills/` holds notes on Skills that fit Cursor—not a random file dump, but enough to see **what it solves, when to use it, and whether it belongs in your flow**.

## Layout

```text
skills/
├── recommended/     # External Skills we vetted
├── custom/          # Custom or adapted drafts
└── README.md
```

## What we include

- Prefer: tried, reusable content
- Required: source, scenario, how to use
- Discouraged: files with no context

## Suggested fields per entry

Name, source or author, scenario, core capabilities, usage, known limits.

## Maintenance

When adding entries, add a short “why this matters”—do not make readers guess from filenames alone.

See [skill development](../docs/skill-development.md) for fuller conventions.

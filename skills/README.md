# Skills 索引

`skills/` 用来放「适合在 Cursor 里用的 Skill」说明：不单堆文件，而是让人快速知道**解决啥、啥时用、值不值得进自己的工作流**。

## 目录

```text
skills/
├── recommended/     # 外部 Skill，我们筛过觉得可试
├── custom/          # 自建或二次整理的草案
└── README.md
```

## 收录标准

- 优先：已试过、能复用
- 必须：来源、场景、怎么用
- 不鼓励：只有文件、没有背景

## 每篇建议包含

名称、来源或作者、场景、核心能力、用法、已知限制。

## 维护

往里加内容时顺手写几句「为什么值得看」，别让人只看文件名猜。

更细的约定见 [Skills 开发说明](../docs/skill-development.md)。

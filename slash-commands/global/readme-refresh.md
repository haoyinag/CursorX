# 刷新 README 或文档说明

目标：根据当前代码、目录结构或最近改动，更新目标 README / 文档段落，使其更准确、更易读、更接近真实使用方式。

用户可以在命令后指定文档目标，例如：

- `/readme-refresh`
- `/readme-refresh packages/cursorx-cli/README.md`
- `/readme-refresh refresh quick start and install verification section`

其中 `${user_input}` 可以是目标文件、目标章节或你希望强化的文档重点。

## 必须遵守

1. 先确认目标文档；如果 `${user_input}` 没有给出明确目标，优先从当前改动最相关的 README 或 `docs/` 文件开始。
2. 更新文档前，必须同时阅读：
   - 目标文档
   - 对应代码或目录结构
   - 相关差异（如果当前工作区有改动）
3. 文档内容必须以仓库真实状态为准，不允许编造未存在的命令、脚本或目录。
4. 优先做增量更新，除非原文结构已经明显失效，不要整份重写。
5. 如果存在不确定信息，先指出假设或需要确认的点。

## 执行步骤

1. 判断本次应更新的文档目标。
2. 阅读目标文档和相关代码、索引、目录说明。
3. 如果当前工作区已有改动，再结合 `git diff` 理解最新变化。
4. 重点检查这些内容是否过时：
   - 安装方式
   - 使用步骤
   - 文件路径
   - 命令示例
   - 推荐入口
5. 用最小必要修改刷新文档内容。

## 输出要求

- 简要说明本次更新了哪些信息
- 如果补了示例或验证步骤，明确指出适用对象
- 如果还有未覆盖的文档缺口，最后列出 1 到 3 条建议

## 适用边界

- 适合 README、快速开始、目录说明、命令说明刷新
- 不适合替代正式技术方案设计文档

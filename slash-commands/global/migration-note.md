# 生成迁移说明

目标：当当前改动涉及命令入口、配置方式、路径、脚本或使用方式变化时，生成一份面向使用者的迁移说明。

用户可以在命令后补充目标读者，例如：

- `/migration-note`
- `/migration-note for users upgrading from the old installer`

其中 `${user_input}` 是目标读者或需要强调的迁移背景，可以为空。

## 必须遵守

1. 先确认改动是否真的涉及迁移影响；不要对纯内部重构硬写迁移说明。
2. 迁移说明必须基于真实 diff 和真实目录变化。
3. 优先说明：
   - 旧方式是什么
   - 新方式是什么
   - 用户需要做什么
   - 不做会有什么影响
4. 如果 `${user_input}` 非空，优先按该受众组织说明。
5. 如果没有明显迁移影响，要明确说明“当前改动不需要额外迁移说明”。

## 执行步骤

1. 优先读取 staged 改动；执行 `git diff --cached --name-only` 和 `git diff --cached`。
2. 如果 staged 为空，再回退到当前工作区 diff。
3. 重点识别这些变化：
   - 文件重命名
   - 命令入口变化
   - 安装方式变化
   - 配置路径变化
   - 用户操作步骤变化
4. 仅当存在实际迁移影响时，输出迁移说明。

## 输出格式要求

- `Who is affected`
- `What changed`
- `What to do`
- `Verification`

如果当前无需迁移，输出一句简洁结论并说明原因。

## 适用边界

- 适合安装方式变化、命令重命名、配置迁移、目录重构说明
- 不适合替代完整发布日志

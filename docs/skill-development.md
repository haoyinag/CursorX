# Skills 目录说明

`skills/` 用于整理与推荐适合 Cursor 的 Skills。重点在「能读懂、能评估、能复用」，而不是堆文件。

## 收录方向

优先两类：

- 经实践验证、值得推荐的外部 Skills  
- 面向明确场景的自建 Skills  

## 提交时建议写清

无论推荐还是自建，建议至少包含：名称、来源或作者、适用场景、核心能力、用法、已知限制。

## 目录

- `skills/recommended/`：外部来源，经筛选与说明的推荐项  
- `skills/custom/`：仓库内自定义整理的 Skills  

## 收录门槛

建议在满足以下条件时再收录：能说明解决什么问题、适合谁/什么工作流、不依赖不透明私有上下文、有最基本使用说明。

## 贡献时多写几句

不要只丢文件；补充：为何推荐、与仓库内其他 Skill 的差异、适合的阶段（入门、重构、审查、写文档等）。

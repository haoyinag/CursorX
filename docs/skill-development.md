# Skills 开发说明

`skills/` 目录用于整理和推荐适合 Cursor 的 Skills。这里更强调“可理解、可评估、可复用”，而不是简单收集文件。

## 收录目标

优先收录这两类内容：

- 已被实践验证、值得推荐的外部 Skills
- 围绕明确场景设计的自定义 Skills

## 建议的提交信息

无论是推荐还是自定义，都建议至少补齐这些信息：

- Skill 名称
- 来源或作者
- 适用场景
- 核心能力
- 使用方式
- 已知限制

## 目录建议

- `skills/recommended/`：外部来源、经过筛选和说明的推荐项
- `skills/custom/`：项目内自定义整理的 Skills

## 收录标准

建议只有在满足下面条件时才收录：

- 能清楚描述它解决什么问题
- 能说明适合什么用户或工作流
- 不依赖不透明的私有上下文
- 有最基本的使用说明

## 贡献建议

提交 Skill 时，不要只提交文件本身，还应补充：

- 为什么推荐它
- 与仓库中其他 Skill 的区别
- 适配的使用阶段，例如入门、重构、审查、文档生成

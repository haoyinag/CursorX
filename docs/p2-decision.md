# P2 当前决策

这份文档记录 `CursorX` 当前对 `P2` 的正式判断，避免后续重复讨论“现在到底该不该切形态”。

## 当前结论

当前进入 `P2` 的方式应为：

- 开始评估
- 暂不切换主产品形态
- 暂不拆仓
- 暂不把 Plugin / Marketplace 作为当前主线

## 推荐路线

当前默认采用：

- 保持单仓
- 强化发布治理
- 持续观察 CLI 是否开始形成独立产品节奏

对应到 [`p2-options.md`](./p2-options.md) 中，等价于：

- 路线 A 为当前主线
- 路线 B 为备选演进方向
- 路线 C 为探索项

## 触发重新决策的信号

当出现下面任一情况时，建议重新评估并可能转向路线 B：

1. `cursorx-cli` 的发版频率显著快于内容仓
2. 包级 changelog、release notes、issue 已经明显独立
3. 用户开始明确只关注安装器，而不关心内容仓
4. 单仓同步 README、索引、版本信息的成本明显升高

当出现下面信号时，才建议认真进入路线 C 的探索：

1. 用户对一键安装和编辑器内发现能力的诉求明显上升
2. CLI 开始成为安装体验的瓶颈，而不是优势
3. Plugin / Marketplace 能力边界已足够清晰

## 当前不做的事

- 不为了“进入新阶段”而强行拆仓
- 不因为 CLI 已发布就默认它已经是独立产品
- 不在证据不足时切到 Plugin / Marketplace 主叙事

## 下一步建议

当前更合理的动作不是继续大规模扩内容，而是：

1. 观察一段时间发布节奏
2. 持续记录用户可感知变化与版本变化
3. 当触发条件明显出现时，再做一次正式的结构决策

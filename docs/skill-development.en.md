# Skills Directory — Maintainer Notes

`skills/` curates and recommends Skills for Cursor. The bar is “understandable, assessable, reusable”—not file hoarding.

## What to include first

- Externally sourced Skills that are battle-tested and worth recommending  
- Custom Skills built around clear scenarios  

## What to document on submission

For both recommendations and custom work, cover at least: name, source or author, scenario, core capabilities, how to use it, known limitations.

## Layout

- `skills/recommended/`: external sources, vetted and explained  
- `skills/custom/`: in-repo curated Skills  

## Inclusion bar

Add entries only when you can state: the problem it solves, who or which workflow it fits, that it does not rely on opaque private context, and there is minimal usage guidance.

## When contributing, add context

Do not submit files alone. Also explain why it is recommended, how it differs from other Skills in the repo, and which phase it fits (onboarding, refactor, review, docs, etc.).

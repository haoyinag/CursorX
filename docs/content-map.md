# 内容地图

**语言 / Language：** [English](content-map.en.md)

回答一件事：进 `CursorX` 以后，**先去哪一层**。

## 要找完整工作流

看：

- [`workflows/README.md`](../workflows/README.md)  
- [`workflows/catalog.md`](../workflows/catalog.md)  

适合：想按场景找「我要做 XX 该怎么走」——提交、PR、发版、排查、交接等。工作流会告诉你该装哪些命令、按什么顺序用。

## 要装完就用

看：

- [`slash-commands/README.md`](../slash-commands/README.md)  
- [`slash-commands/catalog.md`](../slash-commands/catalog.md)  
- [`packages/cursorx-cli/README.md`](../packages/cursorx-cli/README.md)  

适合：在 Cursor 里用 `/命令名`；区分 `global` / `project`；用 `cursorx install`、`verify`、`doctor` 做安装和自检。

## 要找编辑器动作素材

看：

- [`commands/README.md`](../commands/README.md)  
- [`commands/catalog.md`](../commands/catalog.md)  

适合：Quick Chat、命令面板、聊天面板等 editor command；参考 `.json` 怎么组织。

注意：这里是素材/参考层，**不是**主安装入口。

## 要补协作习惯、提问方式

看 [`tips/README.md`](../tips/README.md)。

## 要摘配置片段

看 [`configs/README.md`](../configs/README.md)。

## 要评估或设计 Skills

看：

- [`skills/README.md`](../skills/README.md)  
- [`skill-development.md`](./skill-development.md)  

## 第一次来：最短路径

1. [`workflows/README.md`](../workflows/README.md) — 按场景找工作流  
2. [`getting-started.md`](./getting-started.md) — 装好第一批命令  
3. 按需：  
   - 完整工作流 → `workflows/`  
   - 单条命令 → `slash-commands/` + `cursorx-cli`  
   - 素材 → `commands/`  
   - 方法 → `tips/`  
   - 配置 → `configs/`  
   - Skill → `skills/`

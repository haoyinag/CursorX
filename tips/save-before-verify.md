# 先保存再跑检查

## 什么时候用

- 要跑 lint、测试、构建或发布前检查
- 未保存导致结果忽对忽错
- 想把「验证前收尾」养成肌肉记忆

## 怎么做

1. 跑检查前先把改动的文件存盘。
2. 再跑 lint、类型检查、测试或打包检查。
3. 结果怪时先看磁盘是否和编辑器一致。
4. 提交、发布或切任务前重复这一步。

## 能换来什么

- 少遇「明明改了但结果不对」
- 对文档、配置、命令文件同样成立
- 和 staged review、发布清单很搭

## 可搭配

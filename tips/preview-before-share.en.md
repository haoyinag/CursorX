# Preview before you share

## When

- You just edited README, CHANGELOG, or other docs
- Opening a PR, shipping a release, or sending docs to others
- You do not want link, heading, or layout issues discovered after sharing

## Steps

1. After writing, pause before commit or forward.
2. Open Markdown preview and check headings, lists, code fences, and link text.
3. If this lands in release notes or changelog, skim tone for readers.
4. Then save, validate, and share.

## What you get

- Fewer “content is right but formatting hurts” moments
- Especially helpful when README and release notes change often

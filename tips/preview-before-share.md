# 对外发之前先预览

## 什么时候用

- 刚改完 README、CHANGELOG 或说明文
- 要开 PR、发版或把文档发给别人
- 不想分享后才暴露链接、层级或排版问题

## 怎么做

1. 写完别立刻提交或转发，先停一下。
2. 打开 Markdown 预览，看：标题层级、列表与代码块、链接文案是否自然。
3. 若会进 release notes 或 changelog，再扫一眼语气是否面向读者。
4. 最后再保存、校验、分享。

## 能换来什么

- 少遇「内容对但排版糟」
- 常维护 README、发布说明的仓库尤其有用

## 可搭配

- `open-markdown-preview`
- `/readme-refresh`
- `/release-notes`

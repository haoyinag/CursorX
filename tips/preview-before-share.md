# 分享前先做一次预览

## 适用场景

- 你刚改完 README、CHANGELOG 或说明文档
- 你准备发 PR、发版或把文档贴给别人
- 你不希望链接、层级或排版在分享后才暴露问题

## 具体步骤

1. 写完文档后，不要立即提交或复制给别人。
2. 先打开 Markdown 预览，看：
   - 标题层级是否清晰
   - 列表和代码块是否正常
   - 链接文本是否自然
3. 如果文档会进入 release notes 或 changelog，再检查语气是否面向读者。
4. 最后再做保存、校验和分享动作。

## 效果说明

- 能减少“文档内容对，排版却很乱”的问题
- 特别适合高频维护 README 和发布说明的仓库
- 也有助于维持仓库的利他属性

## 延伸建议

- 可配合 `open-markdown-preview`
- 可配合 `/readme-refresh`
- 可配合 `/release-notes`

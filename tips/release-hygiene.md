# 发版前先做一次发布卫生检查

## 适用场景

- 你准备发布 npm 包、命令集或一批可复用内容
- 你担心“代码有了，但说明和升级路径没跟上”
- 你希望把发布动作变成稳定流程，而不是临时补漏

## 具体步骤

1. 先确认这次发布是否真的有用户可感知变化。
2. 检查版本号、README、CHANGELOG、安装示例是否同步。
3. 检查是否需要：
   - release notes
   - changelog 条目
   - upgrade checklist
4. 再做一次最小安装验证，确认用户按文档操作能真正跑通。

## 效果说明

- 能减少“包发了，但文档还是旧的”这类问题
- 更适合持续发布节奏
- 能显著提升仓库的利他属性

## 延伸建议

- 可配合 `/release-notes`
- 可配合 `/changelog-entry`
- 可配合 `/upgrade-checklist`

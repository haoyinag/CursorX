# 长对话该收尾就收尾

## 什么时候用

- 单会话已经很长
- 模型开始重复、跑偏或丢重点
- 要从一个任务切到另一个

## 怎么做

1. 在当前会话里让模型先给短结论：做了什么、还剩什么、关键文件或风险。
2. 新开一个干净上下文。
3. 把上面三类信息贴过去，再加一句新目标。
4. 别整段历史搬运；优先结论、约束、路径。

## 能换来什么

- 长会话噪声少堆一点
- 适合阶段切换：写完再 review、修完再补文档等
- 大仓库更明显

## 可搭配

- 常切上下文可把「任务交接摘要」做成固定模板
- 若已是全新问题，新开会话往往比硬续旧线程稳

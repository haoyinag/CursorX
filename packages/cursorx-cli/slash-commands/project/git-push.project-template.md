# Git 一键提交并推送（项目模板）

这个模板用于项目级命令，不建议直接作为通用全局命令发布。

你可以基于它增加：

- 默认排除文件
- 某些目录必须拆分提交的规则
- 仓库专属的提交前检查
- 更严格的提交信息约束

## 可定制项示例

### 默认排除模板

每次执行时，先自动从暂存区移除以下文件，再进入勾选步骤：

- `<replace-with-project-specific-path-a>`
- `<replace-with-project-specific-path-b>`

说明：

- 仅在文件已被 staged 时执行 `git restore --staged <file>`。
- 若文件不存在或未改动，忽略即可，不报错中断。

### 目录拆分强校验

如果你的仓库存在必须独立提交的目录，例如：

- `android/`
- `ios/`
- `libs/`

可以在最终 staged 文件确定后增加规则：

- 若这些目录与其它目录混提，则停止并提示拆分提交。

## 推荐做法

1. 复制通用版 [`../global/git-push.md`](../global/git-push.md)
2. 再把项目特有规则补到你的项目命令中
3. 将最终版本放到当前仓库的 `.cursor/commands/` 下使用

这样可以保留公共命令的可复用性，同时允许项目内收紧流程。
